## 管理员酒店审核/发布/下线接口文档编写计划

### 文档结构
在 `PC-api-document3.md` 中创建管理员端 API 文档，包含以下模块：

### 1. 审核列表管理
- **1.1 获取审核列表** - `GET /admin/hotel/audit-list` - 分页查询待审核酒店（支持状态筛选、时间范围、关键词搜索）
- **1.2 获取审核统计** - `GET /admin/hotel/audit-stats` - 今日待审核、已通过、已下线数量
- **1.3 获取酒店详情** - `GET /admin/hotel/detail/:id` - 查看酒店详细信息（用于审核）

### 2. 审核操作
- **2.1 审核通过** - `POST /admin/hotel/approve/:id` - 通过审核
- **2.2 审核拒绝** - `POST /admin/hotel/reject/:id` - 拒绝审核（需填写原因）
- **2.3 批量审核** - `POST /admin/hotel/batch-audit` - 批量审核（通过/拒绝）

### 3. 发布/下线管理
- **3.1 发布酒店** - `POST /admin/hotel/publish/:id` - 将已通过的酒店发布到移动端
- **3.2 下线酒店** - `POST /admin/hotel/offline/:id` - 下线酒店（需二次确认）
- **3.3 恢复酒店** - `POST /admin/hotel/restore/:id` - 恢复已下线的酒店
- **3.4 批量发布/下线** - `POST /admin/hotel/batch-publish` - 批量发布/下线

### 4. 审核日志
- **4.1 获取审核日志** - `GET /admin/hotel/audit-log/:id` - 查看酒店最近一次审核记录

### 数据字段设计
**审核列表字段**：hotel_id, hotel_name_cn, submitted_at, submitted_by, status, status_text, audited_at, audited_by, reject_reason
**审核统计字段**：pending_count, approved_count, offline_count
**审核日志字段**：audit_id, hotel_id, hotel_name, auditor, audited_at, result, result_text, reject_reason

### 文档格式
遵循统一的格式标准：接口地址 → 描述 → 请求参数 → 请求体 → 字段说明 → 响应示例 → 响应字段说明 → 错误码 → curl 示例

所有接口均需在请求头中携带 `Authorization: Bearer {token}`
