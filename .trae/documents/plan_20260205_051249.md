## 手机端注册页 API 接口文档编写计划

### 文档结构
在 `M-api-document1.md` 中创建手机端注册页 API 文档，包含以下模块：

### 1. 注册管理
- **1.1 发送验证码** - `POST /mobile/auth/send-code` - 发送手机验证码（60秒冷却）
- **1.2 校验手机号** - `POST /mobile/auth/check-phone` - 校验手机号是否已注册
- **1.3 手机号注册** - `POST /mobile/auth/register` - 手机号注册（验证码+密码+协议）

### 2. 第三方登录/注册
- **2.1 微信注册** - `POST /mobile/auth/wechat-login` - 微信授权注册
- **2.2 支付宝注册** - `POST /mobile/auth/alipay-login` - 支付宝授权注册

### 3. 通用接口
- **3.1 获取用户协议** - `GET /mobile/agreement/user` - 获取用户协议
- **3.2 获取隐私政策** - `GET /mobile/agreement/privacy` - 获取隐私政策

### 数据字段设计
- 手机号注册：phone（带+86前缀）、code（6位数字）、password（6-16位）、agreed（boolean）
- 第三方注册：open_id（微信）、auth_code（支付宝）、nickname（昵称）
- 验证码限制：expires_in（60秒）

### 错误码设计
- 3001: 手机号格式不正确
- 3002: 验证码发送频率限制
- 3003: 验证码错误或已过期
- 3004: 未同意用户协议
- 3005: 手机号已注册
- 3006: 密码格式不正确
- 3007: 验证码发送失败
- 3008: 第三方授权失败

### 文档格式
遵循统一的格式标准：接口地址 → 描述 → 请求参数 → 请求体 → 字段说明 → 响应示例 → 响应字段说明 → 错误码 → curl 示例
