## API 文档修改计划

### 问题1：价格字段改为房型对应价格
**修改内容：**
- 将 `price`（单个数字）改为 `room_prices`（对象数组）
- 每个房型对应一个价格
- 数据结构示例：
```json
"room_prices": [
  {
    "room_type": "大床房",
    "price": 299.00
  },
  {
    "room_type": "双床房",
    "price": 329.00
  }
]
```
- 移除独立的 `room_types` 字段（因为已包含在 `room_prices` 中）
- 涉及接口：创建酒店、保存草稿、更新酒店、获取详情、获取列表

### 问题2：nearby_info 和 promotion_scenario 为可选字段
**说明：**
- 这两个字段在文档中已标注为"否"（非必填）
- 在某些响应示例中未展示是因为：
  - 草稿保存时可能未填写这些字段
  - 列表接口可能不返回这些详细信息
- **无需修改**，当前设计合理

### 问题3：根据用户角色返回不同的酒店列表
**修改内容：**
- 在"获取酒店列表"接口中增加权限说明
- **商户角色**：只返回当前用户创建的酒店列表
- **管理员角色**：返回所有酒店列表
- 在响应示例中增加说明：
  ```json
  // 商户视角：仅包含自己的酒店
  // 管理员视角：包含所有酒店
  ```
- 在字段说明中增加 `created_by` 字段（创建人ID）
- 在错误码中增加权限相关说明

### 修改范围
- 2.1 创建酒店信息
- 2.2 保存草稿
- 2.3 获取酒店列表（增加权限说明）
- 2.4 获取酒店详情
- 2.5 更新酒店信息
- 2.7 获取历史记录（可选）
