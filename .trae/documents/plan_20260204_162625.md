## 酒店信息管理接口文档编写计划

### 文档结构
在 `PC-api-document2.md` 中创建完整的酒店信息管理 API 文档，包含以下模块：

### 1. 酒店信息管理接口
- **4.1 创建酒店信息** - `POST /hotel/create` - 提交审核
- **4.2 保存草稿** - `POST /hotel/draft` - 保存未完成的信息
- **4.3 获取酒店列表** - `GET /hotel/list` - 分页查询酒店列表
- **4.4 获取酒店详情** - `GET /hotel/detail/:id` - 根据ID获取详情
- **4.5 更新酒店信息** - `PUT /hotel/update/:id` - 编辑已录入的酒店
- **4.6 删除酒店** - `DELETE /hotel/delete/:id` - 删除酒店信息
- **4.7 获取历史记录** - `GET /hotel/history/:id` - 获取最近3次修改记录

### 2. 地图相关接口
- **5.1 地址逆地理编码** - `GET /map/geocode/reverse` - 经纬度转地址（高德地图API）
- **5.2 地址地理编码** - `GET /map/geocode` - 地址转经纬度

### 3. 审核相关接口
- **6.1 提交审核** - `POST /hotel/submit/:id` - 将草稿提交审核
- **6.2 获取审核状态** - `GET /hotel/audit-status/:id` - 查询审核状态

### 数据字段设计
**必须维度**：hotel_name_cn, hotel_name_en, formatted_address, address_component, star_rating, room_types, price, opening_date
**可选维度**：nearby_info, promotion_scenario
**状态字段**：status（draft/pending/approved/rejected）

### 文档格式
遵循统一的格式标准：接口地址 → 描述 → 请求参数 → 请求体 → 字段说明 → 响应示例 → 响应字段说明 → 错误码 → curl 示例

所有接口均需在请求头中携带 `Authorization: Bearer {token}`
