# Yisu Hotel 移动端“全部订单”快捷页面前端流程文档

本文档描述了移动端“订单”快捷页面的业务逻辑。该页面作为“我的”页面订单管理功能的简化快捷入口，仅展示用户的所有订单。

---

## 1. 页面概述
- **功能定位**: 快捷查看用户所有历史订单的列表页。
- **入口**: 通常位于底部导航栏的“订单”标签。

## 2. 订单列表查询逻辑
- **接口调用**: 调用 [Booking_API: 2.1 查询订单列表](file:///d:/Yisu/Yisu-Hotel-backend/docs/mobile/Booking_API.md#L25)。
- **参数映射**: 
    - `status`: 不传该参数或传空字符串 `""`，以获取所有状态的订单。
- **分页处理**: 支持瀑布流滚动加载，通过 `page` 和 `pageSize` 进行分页。

## 3. 交互与跳转逻辑
- **查看详情**: 
    - **操作**: 点击列表中的任意订单卡片。
    - **跳转**: 进入 **订单详情页面**，并传递 `order_id` 参数。
    - **接口逻辑**: 调用 [Booking_API: 2.2 查询订单详情](file:///d:/Yisu/Yisu-Hotel-backend/docs/mobile/Booking_API.md#L110)。
- **跳转酒店详情**: 
    - **操作**: 在订单详情页点击酒店名称。
    - **逻辑**: 携带 `hotel_id` 跳转至 **酒店详情页面**。

---

## 4. 与“我的”页面订单管理的关系
该页面的核心逻辑与 [我的页面.md: 2. 订单管理流程](file:///d:/Yisu/Yisu-Hotel-backend/docs/mobile/我的页面.md#L66) 完全一致，区别在于：
1. **无分类筛选**: 该页面不提供状态筛选标签（待付款、已付款等），仅固定展示“全部订单”。
2. **快捷入口**: 旨在为用户提供更直接的订单查看路径。
