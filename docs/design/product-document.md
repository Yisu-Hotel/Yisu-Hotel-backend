# 产品文档
易宿酒店预订平台 - 移动端用户端产品文档
一、产品信息
1.1 产品名称
易宿酒店预订平台（移动端用户端）
1.2 产品定位
面向旅游出行、商务出差、探亲访友等场景的移动端酒店预订工具，通过简洁高效的交互流程，帮助用户快速完成酒店查询、筛选、详情查看等操作，精准匹配用户需求与酒店资源，提供便捷的预订前全流程服务。
1.3 核心目标
- 简化酒店预订前的操作流程，降低用户决策成本；
- 全面呈现酒店关键信息，助力用户做出合理选择；
- 适配移动端操作习惯，提升用户使用体验。
1.4 目标用户
- 有酒店预订需求的个人用户（涵盖旅游、商务、探亲等场景）；
- 习惯使用移动端（手机）快速完成查询、筛选等操作的用户；
- 对酒店位置、价格、星级、房型、设施等有特定要求的精准需求用户。
二、核心页面详情
2.1 酒店查询页（首页）
入口
移动端 App / 小程序启动后默认进入；
其他页面通过底部导航栏 “首页” 按钮返回进入。
展示模块
- 顶部 Banner：展示酒店推广广告图及简短宣传文案（如 “春节特惠，低至 8 折”），图片适配移动端屏幕尺寸；
核心查询区域：
- 当前地点：显示默认城市或定位城市，右侧带 “切换” 图标；
- 关键字搜索框：提示文案 “输入酒店名称 / 品牌 / 位置”，带搜索图标；
- 入住日期选择框：默认显示 “今天 - 明天 共 1 晚”，带日历图标；
- 筛选条件栏：包含 “星级”“价格”“设施” 等基础筛选入口；
- 快捷标签区：横向滚动展示 “亲子友好”“免费停车场”“含早餐”“豪华型” 等高频属性标签；
- 查询按钮：位于查询区域底部，蓝色背景、白色文字，突出显示 “查询” 字样。
关键操作
- Banner 点击：点击广告 Banner 直接跳转至对应酒店的详情页；
- 定位操作：自动获取用户当前城市（需申请位置权限），用户拒绝后可手动输入城市名称；
- 关键字搜索：输入内容后，点击搜索框内图标或键盘回车触发搜索，也可点击查询按钮跳转；
- 日期选择：点击日期框展开日历组件，支持选择入住 / 离店日期（离店日期需晚于入住日期），提供 “1 晚”“2 晚” 等快捷选择；
- 筛选操作：点击筛选条件栏进入详细筛选页，勾选标签后实时关联查询结果；
- 查询跳转：点击查询按钮，携带所有筛选条件跳转至酒店列表页。
其他功能
- 权限处理：申请位置权限时，弹窗说明 “为精准推荐酒店，需获取您的位置信息”，提供 “允许”“拒绝” 选项；
- 空态处理：无 Banner 广告时显示默认风景占位图；无筛选结果时，列表区域提示 “暂无符合条件的酒店”，并提供 “重置筛选” 按钮；
- 异常处理：定位失败时，当前地点显示 “请选择城市”，并提示 “定位失败，请手动选择”；网络异常时，显示 “网络不佳，请检查连接”。
2.2 酒店列表页
入口
酒店查询页点击 “查询” 按钮跳转进入；
酒店详情页通过 “返回列表” 按钮返回进入（保留历史筛选条件）。
展示模块
- 顶部核心筛选头：固定在页面顶部，包含当前选择城市、入住 / 离店日期、入住间夜数，右侧带 “筛选”“排序” 图标；
- 详细筛选区域：默认收起，点击筛选头 “筛选” 图标展开，包含价格区间滑块、酒店星级多选、设施服务（免费 WiFi、游泳池、24 小时前台等）、用户评分筛选等维度；
- 酒店列表区：采用卡片式设计，单条酒店卡片包含：酒店缩略图、酒店名称、用户评分、收藏数、地址、实时价格、核心标签（如 “可订”“免费取消”）；
- 加载提示：列表底部显示 “加载中...”“已到底部” 等状态提示。
关键操作
- 筛选条件修改：点击顶部筛选头中的城市、日期等项，可直接修改并实时刷新列表；
- 排序操作：点击 “排序” 图标，弹出排序选项（综合排序、价格升序 / 降序、距离由近及远），选择后刷新列表；
- 上滑加载：用户上滑列表至底部，自动加载下一页酒店数据；
- 酒店跳转：点击任意酒店卡片，跳转至该酒店的详情页；
- 收藏操作：点击酒店卡片右上角收藏图标，登录状态下可快速收藏 / 取消收藏，收藏成功后图标变色。
其他功能
- 下拉刷新：支持下拉刷新列表，更新最新酒店数据及价格；
- 空态处理：无符合筛选条件时，显示 “暂无匹配酒店”，提供 “重置筛选” 按钮；
- 异常处理：网络请求失败时，显示 “加载失败，请下拉刷新重试”；图片加载失败时，显示默认酒店占位图；
- 自定义展示：支持用户长按酒店卡片，选择 “不感兴趣” 隐藏该酒店。
2.3 酒店详情页
入口
酒店查询页 Banner 点击跳转；
酒店列表页点击酒店卡片跳转。
展示模块
- 顶部导航栏：左侧 “← 返回列表” 按钮，中间显示酒店名称（长名称自动换行），右侧 “分享”“收藏” 图标；
- 大图 Banner 区：多张酒店实景图（外观、大堂、房间等），支持左右滑动切换，底部显示图片索引（如 “1/6”）；
- 酒店基础信息区：展示酒店星级标识、用户评分、地址、距离当前定位距离，下方横向排列设施图标（免费 WiFi、停车场、空调等）；
- 日历 + 入住间夜区：包含日历组件入口、入住间夜数显示、参考总价预览（如 “参考总价：¥588 共 1 晚”）；
- 房型价格列表区：按价格从低到高排序，单条房型卡片包含：房型名称、床型、面积、设施说明、实时价格、“查看详情”“立即预订” 按钮。
关键操作
- 返回列表：点击顶部导航栏 “返回列表” 按钮，回到之前的酒店列表页并保留筛选条件；
- Banner 浏览：左右滑动切换图片，双击或捏合可放大查看原图，点击 “关闭” 返回缩略图状态；
- 日期修改：点击日历 + 入住间夜区，重新选择日期，实时更新参考总价；
- 房型查看：点击 “查看详情” 按钮，展开该房型完整信息（如最多入住人数、取消政策、额外服务）；
- 预订操作：点击 “立即预订” 按钮，跳转至预订确认页面（选择入住人、联系方式等）；
- 分享操作：点击顶部 “分享” 图标，支持分享酒店信息至微信、朋友圈、短信等平台；
- 收藏操作：点击顶部 “收藏” 图标，登录状态下收藏 / 取消收藏，与列表页收藏状态同步。
其他功能
- 状态展示：房型参与优惠活动时，显示 “8 折优惠”“套餐立减” 等红色标签；剩余库存较少时，显示 “仅剩 2 间” 提醒；
- 空态处理：当前日期下无可用房型时，显示 “暂无可用房型”，并提示 “可修改日期尝试查询”；
- 异常处理：网络异常时，显示 “加载失败，请下拉刷新”；图片加载失败时，显示房型默认占位图；
- 便捷操作：支持点击地址跳转至地图 App 查看导航路线；支持长按价格标签查看价格明细（如房费、服务费等）。
2.4 预订确认页
入口
酒店详情页点击 “立即预订” 按钮跳转进入。
展示模块
- 订单信息区：展示酒店名称、房型、入住 / 离店日期、入住间夜数、房间数量；
- 入住人信息区：包含姓名、手机号输入框（已登录用户自动填充，支持修改）；
- 备注信息区：可选输入框，提示 “如有特殊需求，请在此备注（如无烟房、高楼层）”；
- 价格明细区：列出房费、服务费、优惠金额、总价，明确展示各项费用；
- 预订按钮：位于页面底部，蓝色背景，显示 “确认预订” 字样；
- 协议勾选区：包含 “我已阅读并同意《预订条款》《取消政策》” 勾选框。
关键操作
- 信息填写：输入或修改入住人姓名、手机号，可选填写备注信息；
协议勾选：勾选协议后，预订按钮才可点击；
- 确认预订：点击 “确认预订” 按钮，提交订单并跳转至支付页面；
- 返回修改：点击页面顶部 “返回” 按钮，回到详情页重新选择房型或日期。
其他功能
- 表单校验：姓名、手机号为空时，提交前提示 “请完善入住人信息”；手机号格式错误时，实时提示 “请输入正确的手机号”；
- 取消政策提示：在价格明细区附近显示 “入住前 1 天 18:00 前取消可免费退款” 等政策说明；
- 异常处理：提交订单失败时，提示 “订单提交失败，请重试”；网络异常时，保留已填写信息，避免重复输入。
2.5 支付页
入口
预订确认页点击 “确认预订” 按钮跳转进入。
展示模块
- 订单概览区：简要展示酒店名称、房型、入住日期、订单总价；
- 支付方式区：列出支持的支付方式（微信支付、支付宝支付），带对应图标；
- 订单编号区：显示订单唯一编号，便于查询；
- 支付按钮：位于页面底部，按选择的支付方式显示对应按钮（如 “微信支付”“支付宝支付”）；
- 倒计时区：显示 “请在 15 分钟内完成支付，超时订单将自动取消” 及倒计时。
关键操作
- 支付方式选择：点击对应支付方式图标完成选择，选中状态高亮显示；
- 发起支付：点击支付按钮，跳转至对应支付 App 或内置支付组件；
- 取消支付：点击页面顶部 “取消支付” 按钮，弹出确认弹窗，确认后取消订单并返回详情页；
- 订单查询：点击 “查看订单” 入口，跳转至我的订单页查看当前订单状态。
其他功能
- 支付结果反馈：支付成功后跳转至 “支付成功” 页面，显示订单信息及 “返回首页”“查看订单” 入口；支付失败时提示 “支付失败，请重新尝试”，提供 “重新支付” 按钮；
- 超时处理：倒计时结束未支付，自动跳转至 “订单已取消” 页面，提示 “支付超时，订单已取消”，支持重新预订；
- 异常处理：网络异常导致支付状态无法同步时，提示 “支付状态查询中，请稍后刷新”。
2.6 注册页
入口
-未登录状态下点击 “我的” 页面的 “登录 / 注册” 按钮跳转；
-预订、收藏等需要身份的操作触发后，自动跳转至注册页。
展示模块
- 顶部标题：居中显示 “注册账号”；
- 注册方式选择区：提供 “手机号注册”“第三方快捷注册（微信 / 支付宝）” 两个选项，以图标 + 文字形式展示；
手机号注册表单：
- 手机号输入框：带 “+86” 国家码前缀，支持自动校验手机号格式；
- 验证码输入框：右侧带 “获取验证码” 按钮；
- 密码输入框：默认隐藏输入内容，带 “显示 / 隐藏” 图标；
- 协议勾选区：包含 “我已阅读并同意《用户协议》《隐私政策》” 勾选框；
- 注册按钮：位于页面底部，默认置灰，勾选协议后变为可点击的蓝色；
- 快捷入口：页面底部显示 “已有账号？立即登录”，点击跳转至登录页。
关键操作
 注册方式切换：点击 “手机号注册”/“第三方快捷注册”，切换对应注册表单；
手机号注册：
- 输入手机号后，点击 “获取验证码” 按钮，触发短信验证码发送（需校验手机号格式）；
- 输入验证码、设置密码（6-16 位，含数字 + 字母）；
- 勾选协议后，点击 “注册” 按钮完成账号创建；
- 第三方快捷注册：点击微信 / 支付宝图标，跳转至对应 App 授权页面，授权完成后自动完成注册并登录。
其他功能
表单校验：
- 手机号格式错误时，实时提示 “请输入正确的手机号”；
- 验证码为空 / 错误时，提交前提示 “请输入正确的验证码”；
- 密码不符合规则时，提示 “密码需 6-16 位，包含数字和字母”；
- 验证码限制：“获取验证码” 按钮点击后进入 60 秒冷却，显示倒计时，冷却期间不可点击；
异常处理：
- 手机号已注册时，提示 “该手机号已注册，请直接登录”，并提供 “去登录” 入口；
- 验证码发送失败时，提示 “验证码发送失败，请稍后重试”；
- 第三方授权失败时，提示 “授权失败，请重新尝试”。

入口
注册页完成注册操作后自动跳转。
展示模块

其他功能
- 自动登录：注册成功后自动完成账号登录，状态同步至全平台；
- 引导提示：首次注册用户，页面显示 “首次预订可享新人立减 10 元” 的优惠提示，点击可查看优惠规则。

易宿酒店预定平台（PC端管理系统）产品文档
一、产品信息
1.1 产品定位
易宿酒店预定平台PC端管理系统是面向酒店商户与平台管理员的后台管理工具，专注于酒店信息的录入-审核-发布全流程管理，为商户提供标准化信息维护能力，为管理员提供高效的内容审核与管控能力，是移动端用户端酒店信息展示与预订服务的核心支撑。
1.2核心价值
- 为商户：提供标准化信息录入模板，降低信息维护成本，减少信息填写错误
- 为管理员：提供流程化审核工具，提升平台内容质量，缩短审核周期
- 为平台：实现酒店信息从录入到发布的闭环管理，保障信息与移动端实时同步
二、页面详情
2.1 用户登录/注册页
入口
- 直接访问PC端管理系统域名
- 未登录状态下访问其他功能页面时自动跳转
展示模块
1. 注册界面（默认展示）：
- 角色选择栏：“商户”“管理员”单选按钮（默认选中“商户”）
- 表单区：账号（手机号/邮箱）输入框、密  码输入框（支持显示/隐藏）、验证码输入   框 +“获取验证码”按钮
- 协议区：“我已阅读并同意《用户协议》《隐私政策》”勾选框
- 操作区：“注册”按钮（勾选协议+表单填写完整后可点击）
- 辅助链接：“已有账号？立即登录”
2. 登录界面（点击“立即登录”切换）：
- 表单区：账号输入框、密码输入框（支持显示/隐藏）
- 操作区：“登录”按钮
- 辅助链接：“忘记密码？”“没有账号？立即注册”
关键操作
1. 注册：
- 选择角色→填写表单→勾选协议→点击“注册”→完成账号创建
2. 登录：
- 输入账号密码→点击“登录”→系统自动识别角色并跳转对应首页
3. 忘记密码：
- 输入绑定账号→获取验证码→重置密码
其他功能
- 注册时自动校验账号唯一性（提示“该账号已被注册”）
- 登录失败时精准提示错误原因（如“账号不存在”“密码错误”）
- 验证码获取后倒计时60秒，防止频繁请求
2.2 酒店信息录入/编辑/修改页
入口
- 商户角色登录后，首页直接展示
- 左侧导航栏“酒店信息管理”→“信息录入/编辑”
展示模块
1. 必须维度表单区（红色星标标记）：
- 酒店名：中文输入框+英文输入框（支持双语言显示）
- 酒店地址：文本输入框+“地图选点”按钮（弹窗选择地址）
- 酒店星级：下拉选择框（1-5星）
- 酒店房型：多选框（如“大床房/双床房/套房”）
- 酒店价格：数字输入框（支持小数，默认保留2位）
- 酒店开业时间：日期选择器（支持选择过去日期）
2. 可选维度折叠区（默认收起）：
- 周边信息：多行文本输入框（支持输入景点、交通等）
- 优惠场景：文本输入框+下拉选择（如“节日优惠/机票+酒店套餐”）
3. 操作区：
- “保存草稿”按钮（灰色，点击后提示“草稿已保存”）
- “提交审核”按钮（蓝色，必填项填写完整后可点击）
4. 提示区：
- 底部文字：“*为必填项，未填写无法提交审核”
关键操作
1. 信息录入：
- 填写必须维度信息→（可选）展开填写可选维度→点击“提交审核”→发送至管理员审核队列
2. 信息编辑：
- 选择已录入的酒店→修改信息→点击“保存”→实时更新数据
3. 草稿保存：
- 未完成填写时，点击“保存草稿”→保留当前内容（后续可继续编辑）
其他功能                                                
- 地图选点：点击按钮后弹出地图弹窗，选点后自动填充地址输入框 
- 表单校验：必填项为空时，“提交审核”按钮置灰并提示“请填写必填项”
- 历史记录：展示最近3次信息修改的时间与修改人
2.3 酒店信息审核/发布/下线列表页
入口
- 管理员角色登录后，首页直接展示
- 左侧导航栏“酒店信息管理”→“审核列表”
展示模块
1. 筛选区：
- 审核状态下拉框（待审核/审核中/通过/不通过）
- 提交时间日期选择器（支持选择时间范围）
- “搜索”按钮
2. 列表区：
- 列：酒店名、提交时间、提交商户、审核状态、操作
- 每行操作栏：“查看详情”“审核”“发布/下线”按钮
3. 数据概览区：
- 顶部卡片：“今日待审核：X条”“已通过：X条”“已下线：X条”
关键操作
1. 信息审核：
- 点击“审核”→弹窗展示酒店信息→选择“通过/不通过/审核中”：
- 不通过：填写原因（必填）→点击“确认”→反馈给商户
- 通过：点击“确认”→信息状态更新为“已通过”
2. 信息发布：
- 对“已通过”的酒店→点击“发布”→提示“发布成功，已同步至移动端”
3. 信息下线：
- 对“已发布”的酒店→点击“下线”→二次确认弹窗→点击“确认”→移动端隐藏该酒店
其他功能
- 批量操作：勾选多条酒店→点击顶部“批量审核/发布/下线”→统一执行操作
- 审核日志：每条信息展示最近一次审核的“审核人+时间+结果”
- 信息恢复：对“已下线”的酒店→点击“恢复”→重新发布至移动端
2.4 审核状态查询页（商户端）
入口
- 商户角色左侧导航栏“酒店信息管理”→“审核状态”
展示模块
1. 列表区：
- 列：酒店名、提交时间、审核状态、审核结果、操作
- 操作栏：“查看原因”（仅“不通过”状态显示）
2. 提示区：
- 状态说明：“待审核：等待管理员处理；审核中：管理员正在审核；通过：可发布；不通过：需修改后重新提交”
关键操作
1. 查看审核结果：
- 对“不通过”的酒店→点击“查看原因”→弹窗展示审核不通过的具体说明
2. 重新提交：
- 对“不通过”的酒店→点击“编辑”→修改信息后重新提交审核
其他功能
- 状态筛选：支持按审核状态快速过滤列表
- 自动刷新：每30秒自动刷新列表，展示最新审核状态
