# Rules

## Git

- 永远不要在main分支上直接提交代码。所有更改必须通过Pull Request进行审查和合并。
- 实现某个功能时或修复bug时，请创建一个新的分支，分支名称应简洁且描述性强，例如：feature/add-login、bugfix/fix-crash。
- 发起Pull Request时，请确保你的代码已经过测试，发起后尽快@其他人给你review。
- 在你的提交信息中使用清晰且描述性的语言，遵循约定的提交信息格式（例如：feat, fix, docs, style, refactor, test, chore）。
- 不要把未经review的代码合并到主分支。
- 不要把未经测试的代码合并到主分支。
- 尽量不要使用`git add .`，在每次提交前，仔细检查你要提交的文件。
- 禁止使用`git push --force`，除非你非常确定你在做什么。

## AI

- 支持使用AI工具写代码，但是你要理解AI生成的代码，并对其进行审查和测试。

## 代码风格

- 确保每个接口的签名都有清晰的注释，说明其用途和参数。如果接口较复杂，建议提供使用示例，然后写入文档中。
